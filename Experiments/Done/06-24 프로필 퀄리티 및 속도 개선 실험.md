## 실험 요약

- 대상: 신규 유저 및 Meta 유입 유저.
- 기존 방식: 유저 선택지를 받아 이미지가 기계적으로 조립됨.
- 새로운 방식: 유저 선택지를 자연어로 정리한 뒤, Grok이 이미지 프롬프트를 만들고, 고퀄리티 LoRA를 적용해 이미지 생성.
- 기대 효과: 프로필 이미지 퀄리티 개선 + 생성 속도 유지.
- 상세 구현: [[docs/06-24 프로필 퀄리티 및 속도 개선 실험 스펙]].

## 실험 대상

- 신규 유저.
- Meta 광고 유입 유저.
- create 온보딩에서 프로필 이미지를 처음 보는 유저.

## 비교 방식

| Arm | 내용 |
| --- | --- |
| Control | 기존 프로필 이미지 생성 방식 |
| Treatment | 새 프로필 이미지 생성 방식 |

## 검증 기준

| 기준 | 확인 |
| --- | --- |
| 같은 입력 | Baseline 10의 선택지 input 동일 사용 |
| 결과 이미지 | 기존 결과와 새 결과를 나란히 비교 |
| 퀄리티 | 얼굴, 의상, 배경, 분위기가 선택지 의도와 맞는지 확인 |
| 속도 | duration 기록. 5초 기준은 제외 |
| 저장 | 새 이미지 R2 URL 존재 |
| manifest | manifest 링크 존재 |
| 제외 | 첫 메시지, 대화 맥락 이미지, 내부 프롬프트 평가는 제외 |

## 테스트 방법

- 아래 Baseline 10을 기준으로 확인.
- 각 row에서 기존 이미지와 새 이미지를 비교.
- 새 이미지가 유저 선택지 의도를 더 잘 보여주는지 확인.
- 생성 시간은 기록만 하고 pass/fail 기준으로 쓰지 않음.
