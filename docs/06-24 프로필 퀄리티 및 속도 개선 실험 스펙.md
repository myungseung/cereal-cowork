## 입력 기준

- 유일한 입력 변수: create 온보딩에서 유저가 고른 선택지.
- 형식: 한글 comma-separated string.
- 처리: `body.input` string만 허용, `trim()` 후 빈 값 차단.
- 사용 금지: prompt list, response text, api log text.

## Treatment Flow

```txt
create 선택지 input
-> Grok image prompt 생성
-> Replicate image 생성
-> R2 image 업로드
-> R2 manifest 저장
```

## Grok Prompt 생성

- 모델: `x-ai/grok-4.20`.
- 목적: 선택지 키워드를 촬영 가능한 영어 이미지 프롬프트로 확장.
- 출력: JSON/markdown 없이 최종 영어 image prompt만 반환.
- 유지 조건: 현재 Grok 시스템 프롬프트 변경 금지.
