### 첫 메시지

```text
현재
creator device -> body.firstMessage
new device     -> fixed prologue

목표
모든 device -> Chat Orchestrator의 동일 opening result
```

- 캐릭터 설정과 맞는 접점.
- 왜 지금 연락했는지에 대한 궁금증.
- 답하기 쉬운 구체적 질문.
- 다음 대화에서 회수할 훅.
- 즉시 연인·고백 강제 금지.
- 맥락이 충분하면 당일 관계 진전 허용.

### 10턴 Golden Path

| 순서 | 역할 | 필수 결과 |
|---:|---|---|
| 1 | 첫 메시지 | 왜 연락했는지 궁금함 |
| 2 | 첫 답변 | 사용자 답변을 실제 반영 |
| 3 | 관계 탐색 | 캐릭터 설정에 맞는 질문 |
| 4 | 상호성 | 캐릭터도 작은 사실 공개 |
| 5 | 기억 | 앞선 사용자 정보를 자연스럽게 재사용 |
| 6 | 미완성 훅 | 다음 답변이 궁금한 상태 |
| 7 | 이미지 맥락 | 보여줄 이유가 대화에서 발생 |
| 8 | 감정 진전 | 이전 대화 근거로 친밀도 상승 |
| 9 | 영상 맥락 | 핵심 감정·행동을 보고 싶은 상태 |
| 10 | 영상 이후 | 다시 답하고 싶은 훅 |

10턴은 E2E 테스트 시나리오. 제품의 관계 threshold 아님.

## Gate 2 — 이미지

```text
Chat Orchestrator mediaOffer.sceneSlots
        |
        v
sceneSpec
emotion + viewerRole + camera
+ expression + completedAction + bodyLanguage
+ failureConditions
        |
        v
deterministic prompt builder
character + outfit + location + sceneSpec + sharedRules
        |
        v
fixed image input
z-image-turbo + seed + LoRA
        |
        v
R2 image.jpg + manifest.json
        |
        v
user quality gate
```

- 유저 요청을 무조건 이미지화하지 않음.
- 대화와 관계 진전에 필요한 경우만 제안.
- `sceneSlots`를 추적 가능한 `sceneSpec`으로 변환.
- `sceneSpec` + 검증된 고정 block을 순서대로 조합.
- 별도 image-prompt LLM 호출 0.
- 실행 중 자유문장 생성·재해석 0.
- 대화 원문은 `userMessage` / `assistantMessage`로 manifest에 보존.
- 장면 의도는 `emotion` / `viewerRole` / `camera` / `expression` / `completedAction` / `bodyLanguage` / `failureConditions`로 보존.
- 최종 prompt와 `sceneSpec`, seed, model, LoRA를 manifest hash로 검증.
- 사용자 명시 tap 후 200콘 선차감.
- 생성 실패 시 200콘 전액 환불.
- PASS 이미지가 다음 영상의 start image.

```json
  "emotion": "",
  "viewerRole": "",
  "camera": "",
  "expression": "",
  "completedAction": "",
  "bodyLanguage": "",
  "failureConditions": []
}
```

| 단계 | 호출 |

| `sceneSpec` 변환 | 추가 LLM 0, 고정 매핑 |
| 최종 prompt 조립 | 추가 LLM 0, deterministic builder |
| 이미지 생성 | z-image-turbo 1회 |
| manifest 검증 | 추가 LLM 0, hash 비교 |

## Gate 4 — 영상

| 항목 | 값 |
|---|---|
| Model | `prunaai/p-video` base |
| Resolution | 720p |
| Provider 원가 | $0.02/초 |
| Input | Gate 2 PASS image + Gate 3 PASS WAV |
| Duration | audio가 결정, 3–6초 |
| 판매가 | 900콘 |
| Timeout | 60초 |
| 실패 | 900콘 자동 환불 |

첫 launch template 1개:

```text
template_key: talk_closeup
framing: 얼굴과 입이 분명한 상반신
motion: 단일 감정 + 단일 행동
camera: 고정
audio: exact approved spoken line
identity: Gate 2 PASS image
```

제외:

- 복수 행동.
- 장면 전환.
- 큰 카메라 이동.
- 입을 가리는 손.
- 외부 인물·손.
- 전신 복잡 동작.
- 임의 의상·배경 변경.

### 안정성 Gate

| 단계 | 테스트 | 통과 |
|---|---|---|
| 반복성 | 동일 image/audio/template 3회 | 3/3 critical PASS |
| 맥락성 | 서로 다른 spoken line 5개 | 4/5+ PASS |
| Lip sync | 한국어 음절·입·턱 움직임 | 명백한 지연/누락 0 |
| Identity | 얼굴·헤어·의상 | 변형 0 |
| Motion | 단일 행동 | 이상 손·몸 변형 0 |
| Audio | 승인 WAV 유지 | duration/stream PASS |
| 기록 | R2 video/manifest/thumb/history | 필드 누락 0 |

실패 규칙:

- 한 번에 변수 1개만 변경.
- 동일 실패 2회면 template 폐기.
- prompt 무한 수정 금지.
- Kling 자동 fallback 금지.
- PASS 없는 결과 제품 등록 금지.
