| ------ | ---------------------------------- |
| Status | 승인 대기                              |
| Scope  | arin 캐릭터 1개, 데모 데이스                |
| 핵심     | p-video API 연동 + idle ↔ dynamic 전환 |

## Goal

- 텍스트 대화 중 LLM이 "동영상으로 보여줄 순간"을 감지
- p-video API로 비디오 생성 후 idle → dynamic → idle 전환
- 전체 플로우 체크: 채팅 → 하이라이트 감지 → 비디오 생성 → 표시

## 아키텍처

```
┌─────────────────────────────────────────────────────────┐
│                    Video Chat Demo                       │
├─────────────────────────────────────────────────────────┤
│  [Idle State]                                           │
│  ┌─────────────────────────────────────────────────┐   │
│  │  idle video (arin-idle-5s-slight-body-motion)   │   │
│  │  first frame = last frame (반복 재생)            │   │
│  │  백그라운드에서 p-video 생성 대기                  │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  [Chat Interface]                                       │
│  ┌─────────────────────────────────────────────────┐   │
│  │  DMChatView (기존 컴포넌트)                       │   │
│  │  사용자 메시지 → LLM 응답                        │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  [Highlight Detection]                                  │
│  ┌─────────────────────────────────────────────────┐   │

│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  [Seamless Transition]                                  │
│  ┌─────────────────────────────────────────────────┐   │
│  │  idle 비디오 재생 중 p-video 생성 완료           │   │
│  │  → idle 마지막 루프 종료 시 dynamic 비디오 이어붙이기│   │
│  │  → dynamic 비디오 재생 완료 → idle 복귀           │   │
│  │  (로딩 상태 없음, 사용자 인지 불가)               │   │
```

## 핵심 컴포넌트

### 1. Idle Video Player
- 소스: `arin-idle-5s-slight-body-motion.mp4`
- first frame = last frame (반복 재생 시 자연스러운 연결)
- 상태: `idle` | `dynamic`
- 백그라운드에서 p-video 생성 대기 (사용자 인지 불가)

### 2. Chat Interface
- 기존 `DMChatView` 컴포넌트 재사용
- 메시지 전송 시 LLM 응답 + highlight 감지 병렬 처리

### 3. Highlight Detection (LLM Call)
```
입력: 대화 히스토리 (최근 N턴)
출력: {
  isHighlight: boolean,
  videoPrompt: string (if isHighlight),
  context: string (비디오 설명)
}
```

### 4. Video Generation (p-video API)
```
입력: {
  image: first_frame_url (idle last frame),
  prompt: videoPrompt,
  duration: 5,
  resolution: "720p"
}
출력: video_url
```

### 5. State Manager
```
States:
- idle: idle 비디오 반복 재생 + 백그라운드 p-video 생성 대기
- dynamic: 생성된 비디오 1회 재생 → idle 복귀

Transition:
- idle → dynamic: idle 마지막 루프 종료 시 dynamic 비디오 이어붙이기
- dynamic → idle: dynamic 비디오 재생 완료 시
```

## 데이터 흐름

```
1. 사용자 메시지 전송
2. DMChatView → LLM 응답 생성
3. 병렬: Highlight Detection LLM call
4. isHighlight = true?
   ├─ No → 일반 텍스트 응답만 표시
   └─ Yes → p-video API call (백그라운드, idle 비디오 계속 재생)
5. p-video 응답 수신 → ready 상태로 대기
6. idle 비디오 마지막 루프 종료 → dynamic 비디오 이어붙이기
7. dynamic 비디오 재생 완료 → idle 복귀
```
